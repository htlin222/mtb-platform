# 05 · 競賽策略

## 前提:風險已翻面

模組皆為現成、可跑。因此一週後決勝的不是「做不做得出來」,而是**「評審信不信、記不記得住」**。整套策略照這個新現實走。

## 誰在評審,各自吃什麼

**Gladstone(研究機構)** 吃「科學嚴謹度」。你的 PRISMA 系統性回顧當骨幹,把作品從「又一個 ChatGPT 包裝」拉高到「這真能幫我實驗室」。多數隊伍拿不出這層。

**Anthropic** 在意「展示 Claude 的 agentic 能力 + 高風險領域負責任地部署」。多來源子代理編排(NGS+臨床+NCCN+文獻)是單一 prompt 做不出來的真編排;而 provenance、不確定性標記、人工簽核當**核心而非裝飾**——正中他們的價值觀。**把安全當進攻,不是防守;多數隊伍會跳過。**

**真問題本身**有血有肉:小醫院開不起分子腫瘤委員會。評審記得住有真實對象的作品。

## 頭號陷阱:完整系統,全秀就是輸

你手上有 NGS+臨床+NCCN+PubMed+Scopus+PRISMA+報告。正因都能動,最大誘惑是通通 demo。**別。** 全秀 = 沒有記憶點,評審一天看幾十個,只記得住有**單一高峰**的那個。紀律不在「做什麼」,在**「不秀什麼」**——狠心砍到一條故事線。

## 現在才解鎖的贏招:上台跑評審自己的資料

工作量不是問題後,這變成該打的牌。用 Gladstone 提供的真實資料集(T 細胞定序 / DNA 調控 / 蛋白質網路)現場跑出可驗證結果——這一刀把「酷炫包裝」與「這真能用」切開。橋接兩賽道:開發組工具,引擎跑實驗室組等級推理,吃評審的資料。

## 能贏下這組評審的 flex:當場讓他們挑戰

「隨便點一條建議,順著 citation 往下追。」對研究機構,能扛住檢驗的系統值得尊敬;對 Anthropic,這正是負責任部署。讓「懷疑」被介面獎勵——這是你真正的護城河。

## 別藏的差異化:這是飛輪,不是工具

臨床需求 → 研究議程的迴圈(見 `03`),對研究機構評審遠比「又一個臨床 AI」動人。因為 PRISMA 已現成,你能**當場演這個迴圈**,不是用嘴講。

## 反直覺警告:太順反而可疑

Anthropic 的工程師最清楚 agentic 系統多脆——一個滴水不漏、明顯預錄的 demo 會讓他們懷疑你 pre-bake。解法只有一個:**做一件當場、沒排練的事**——接受評審臨時丟的一個變異或問題現場跑。而「敢當場跑」本身,只有系統真的能動的隊伍做得到。這是你獨佔的信任籌碼。

## 結論

既然都能動,**押大的**:真資料 + 開放挑戰 + 飛輪,而不是保守的合成病例。腳本見 `06`。
