# 06 · 三分鐘 Demo 腳本

## 先決:money shot 押哪個(唯一未決)

- **A(建議):吃 Gladstone 真實資料集,現場跑可驗證結果** — 高風險高報酬。
- **B(保守):合成癌症病例 → 現場產出可簽核報告** — 穩、完整,記憶點較弱。

下列腳本以 **A** 撰寫。若選 B,只需替換 0:20–0:50 段(用預備好的合成病例),其餘不變。

## 三分鐘骨架

**0:00–0:20 · 鉤子(一句話講完問題與對象)**
「小醫院開不起分子腫瘤委員會。我們做了一個——而且它跑在你們自己的資料上。」不鋪陳背景,直接進場。

**0:20–0:50 · 一個真實案例進場 → agents 現場並行跑**
把 pipeline 用 checks 樣式攤開(NGS 判讀 / 臨床脈絡 / NCCN / 文獻),讓評審看到編排在動。結束時結構化報告成形。這是「透明過程」的劇場。

**0:50–1:40 · 第一層讀完 → 往下鑽(關鍵信任節拍)**
先在第一層掃完結論,然後挑一條建議,順著 citation 一路追到原始證據 + PRISMA flow。這一刻證明「它站得住」。

**1:40–2:20 · 飛輪(全案最強差異化)**
現場秀:一個證據缺口 → 成為回顧題目 → (現成的)系統性回顧 → 報告升級。一句話收:「臨床需求驅動研究議程。」

**2:20–2:50 · 沒排練的一刻(信任籌碼)**
邀評審當場丟一個變異或問題,現場跑。只有系統真能動的隊伍敢這樣。這是你獨佔的一步。

**2:50–3:00 · 收尾**
一句話講對小醫院的意義 + 兩賽道橋接。結束。

## 現場翻車退路(30 秒)

真資料現場跑有風險。備一份**預先擷取的同案結果**,萬一 live 卡住,坦白切換:「網路/資料當下有狀況,這是我們稍早在同一份資料上跑出的結果」——誠實反而加分,且不打斷故事線。切勿假裝 live。

## 一頁講者小抄(可貼在螢幕邊)

- 開場句先講,不要先自我介紹。
- 全程只走**一條**故事線;忍住不要展開其他模組。
- 「往下鑽」與「開放挑戰」是兩個高峰,其餘壓低。
- 顏色只講臨床意義,不解釋任何技術。
- 若被問技術細節,拉回「這對主治/實驗室意味什麼」。
